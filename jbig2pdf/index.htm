テキスト中心の本のスキャンからjbig2で小さいPDFを作成する　2026年6月8日更新

　

スキャナーで作成したPDFや電子書籍のスクリーンショットで作成した画像から小さいPDFを作成をする。ページ中のテキスト等はjbig2形式でモノクロに圧縮し、写真等はグレイスケールjpeg形式でPDFの元の位置にスタンプとして貼り付ける方針で。

使用するソフトは jbig2enc (別のサイトにもあるが正しく動作しない)、ImageMagick、CPDF、BusyBox for Windows、Python 等。Pythonには pip install で opencv-python と img2pdf パッケージをインストールする。画像のPDF化は img2pdf.py の替わりに CPDF でもできるが作成されるPDFが常に72dpiになってしまうので使えない。

スクリーンショットの自動化方法については書かないので必要であれば何らかの方法でご自分でご用意下さい。

(1) スキャンでPDFが作成されるのであればその中からまず個々の画像ファイルを抽出する。Poppler の pdfimages.exe コマンドを使ったり、私のスキャナーのPDFは 画像梱包 でも抽出できた。

(2) 必要により何らかのツールで画像のサイズ（寸法）を小さくする。変換後の画像形式は png がよい。サイズを小さくすれば作成されるPDFは小さくなるが、透明テキストを付けるのであれば、サイズを小さくし過ぎるとOCRの認識精度は下がってしまう。

(3) 作業フォルダーを作成して中に"pages"というフォルダーを作成してその中にページ順にソートできるファイル名で画像ファイルを入れる。コマンドプロンプトの dir コマンドのソート順が使われる。たとえば"8.png"、"9.png"、"10.png"というファイルがある時Windowsのエクスプローラーはその順でソートするが、dir では"10.png"→"8.png"→"9.png"となることに注意。"08.png"、"09.png"、"10.png"であれば問題ない。

(4) 透明テキストを結合するのであれば「OCRで作成されたPDFから「透明テキストを抽出・可視化」 で透明テキストPDFを作成しておく。OCRで作成するPDFのdpi（解像度）と後述 "make_pdf.sh" 中に設定するdpiとは同一にする必要がある。OCRが作成するPDFのdpiは Poppler の　pdfimages.exe コマンドで

pdfimages -list PDFファイル

と実行すると x-ppi・y-ppi として表示される。
OCRがページごとにPDFを作成するのであれば何かのツールで無劣化結合してから透明テキスト作成バッチファイルを実行する。CPDF コマンドを使用するのであれば、

cpdf -merge -idir 対象PDFフォルダー -o 結合後.pdf

が使える。
ちなみに NDLOCR-Lite ではv1.2.2 (pre-release) 以降からdpiが指定できるようになった。

(5) (3) の作業フォルダーをカレントフォルダーにして make_pdf.sh とget_img_pos.py を置き、以下のコマンドを実行する。その前に make_pdf.sh スクリプト中の *.exe や *.py のパスと冒頭の dpi 変数の値をテキストエディターで書き換える。sh中のパスの区切り文字は"\"ではなく"/"を使う（カレントフォルダーにパスは通っていないのでその場合には ./*.exe とする）。また、magick.exe の引数を変更して写真等を好みの画質にする。（get_img_pos.py の作成は簡単ではなかったので Gemini に作ってもらった。）

busybox ash -c ./make_pdf.sh
（ちなみに busybox の ash は bash の一部構文が使える）

(6) output.pdf が作成される。別途透明テキストを作成していた場合 次のコマンドで結合できる。

cpdf -combine-pages output.pdf 透明テキスト.pdf -o output2.pdf

その後 cpdf -squeeze output2.pdf -o output3.pdf を使うとサイズが更に小さくなるかもしれない。

※ スキャナーが作成したPDFはページごとに画像形式とdpiが異なることがある。

pdfimages -list PDFファイル

で確認できる。本来dpiごとに処理を分けて作るべきだが上記では全ページ同じdpiで処理している。そのためPDFリーダーで100%表示にした時のページの大きさが変わってしまう。画面上で普通に読むだけであればそれでも問題はない。
